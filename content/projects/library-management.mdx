problem: Library staff spent hours manually tracking which books were issued, overdue, or available. Students had no way to search the catalog or check availability without visiting the desk.
solution: Built a self-service catalog with real-time availability, multi-filter search, and automated fine calculation for overdue books. Admin panel handles the full book lifecycle.
result: Streamlined the borrowing process with automated due date tracking and fine calculations. Search response time under 200ms even with large catalogs.
architecture: React SPA with context for state management. Express.js REST API with Mongoose for data modeling. MongoDB text indexes enable fast full-text search. Cron job checks overdue books daily.
challenges:
  - MongoDB text search এর relevance score fine-tune করা — title কে author এর চেয়ে বেশি weight দিতে হয়েছে
  - Book return এর সময় fine calculation logic — weekend, holiday skip করা
  - Large book list এ cursor-based pagination implement করা — offset pagination scale করে না
  - Concurrent borrow request handle করা — race condition avoid করতে atomic transaction use করতে হয়েছে
futurePlans:

  - Integration with national library database
lessons:
  - MongoDB text indexes with weights provide surprisingly good search relevance
  - Designing a clear borrowing state machine prevents bugs
  - Fine calculation logic benefits from being a pure function (easy to test)
