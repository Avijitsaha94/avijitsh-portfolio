problem: Local event discovery is fragmented — hosts have no easy way to monetize gatherings, and users struggle to find and book relevant local events. Existing platforms are either too complex or don't support local payment methods like bKash and Nagad.
solution: Built a modular REST API with 6 domains (auth, users, events, bookings, reviews, payments) supporting three user roles (User, Host, Admin). Integrated SSLCommerz for local payment methods with server-side transaction validation. Built role-specific dashboards with Recharts data visualizations powered by MongoDB aggregation queries. Implemented dynamic search, multi-field filtering, and URL-synchronized pagination using Next.js App Router.
result: Successfully deployed a production-grade platform with a complete CI/CD pipeline — Vercel for frontend, Render for backend, MongoDB Atlas for database. The platform supports the full event lifecycle from discovery to payment confirmation with automated email notifications via Nodemailer.
architecture: Next.js 14 App Router frontend deployed on Vercel. Node.js/Express.js modular REST API deployed on Render. MongoDB Atlas cluster for data. JWT + Google OAuth for authentication. SSLCommerz for payments with server-side validation to prevent fraud. Cloudinary for image storage. API rate limiting with express-rate-limit for production security.
