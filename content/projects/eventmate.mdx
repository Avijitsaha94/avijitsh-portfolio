problem: "Local event discovery is fragmented. Hosts have no easy way to monetize gatherings, and users struggle to find and book relevant local events. Existing platforms do not support local payment methods like bKash and Nagad."
solution: "Built a modular REST API with 6 domains (auth, users, events, bookings, reviews, payments) supporting three user roles (User, Host, Admin). Integrated SSLCommerz for local payment methods with server-side transaction validation. Built role-specific dashboards with Recharts powered by MongoDB aggregation queries."
result: "Successfully deployed a production-grade platform with a complete CI/CD pipeline. Vercel for frontend, Render for backend, MongoDB Atlas for database. Supports the full event lifecycle from discovery to payment confirmation with automated email notifications."
architecture: "Next.js 14 App Router frontend on Vercel. Node.js/Express.js modular REST API on Render. MongoDB Atlas cluster. JWT + Google OAuth for authentication. SSLCommerz for payments with server-side validation. Cloudinary for image storage. API rate limiting with express-rate-limit."
