problem: Building real-time communication is complex — traditional HTTP polling is slow and resource-heavy. Users needed instant message delivery, online/offline presence tracking, image sharing, and secure authentication without relying on third-party auth providers.
solution: Built a WebSocket-based chat system using Socket.io for bi-directional real-time messaging. Implemented custom JWT authentication from scratch, integrated Cloudinary for image uploads, Resend for welcome emails, and Arcjet for API rate limiting. Zustand handles global state on the frontend with a clean React + Tailwind UI.
result: Achieved sub-100ms message delivery latency. The application supports multiple concurrent users with real-time online/offline presence indicators, zero data loss, and clean reconnection handling. Deployed on Sevalla with MongoDB for data persistence.
architecture: React frontend connects via Socket.io to Node.js/Express server. Custom JWT tokens are verified on each socket connection. Messages and user data are stored in MongoDB. Cloudinary handles image storage, Arcjet protects API routes from abuse.
challenges:
  - Custom JWT authentication implement করা third-party provider ছাড়া — token refresh logic সঠিকভাবে handle করতে হয়েছে
  - Socket.io তে real-time online/offline presence tracking — user disconnect হলে room থেকে properly remove করা এবং অন্য users কে notify করা
  - Multiple concurrent users এর সাথে memory leak avoid করা — socket event listener cleanup সঠিকভাবে করতে হয়েছে
  - Arcjet rate limiting integrate করা — legitimate users block না করে spam/abuse prevent করা
  - Cloudinary image upload এর সময় large file handling এবং proper error feedback দেওয়া
